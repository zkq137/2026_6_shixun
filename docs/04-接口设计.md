# 接口设计

## 1. 通用约定

基础路径：

```text
/api
```

统一响应格式：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

分页响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "items": [],
    "total": 100,
    "page": 1,
    "page_size": 10
  }
}
```

常用错误码：

| code | 说明 |
| --- | --- |
| 0 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或 token 无效 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 409 | 业务冲突，例如库存不足 |
| 500 | 系统错误 |
| 600 | AI 服务不可用 |

## 2. 用户认证接口

### POST /api/auth/register

用户注册。

请求：

```json
{
  "username": "alice",
  "password": "123456",
  "phone": "13800000000"
}
```

响应：

```json
{
  "id": 1,
  "username": "alice",
  "nickname": "alice"
}
```

### POST /api/auth/login

用户登录。

请求：

```json
{
  "username": "alice",
  "password": "123456"
}
```

响应：

```json
{
  "access_token": "token",
  "token_type": "bearer",
  "user": {
    "id": 1,
    "username": "alice"
  }
}
```

### GET /api/users/me

获取当前用户信息。

## 3. 商品与分类接口

### GET /api/categories

获取启用状态的分类列表。

### GET /api/products

商品分页列表。

查询参数：

| 参数 | 说明 |
| --- | --- |
| keyword | 搜索关键词 |
| category_id | 分类 ID |
| min_price | 最低价 |
| max_price | 最高价 |
| page | 页码 |
| page_size | 每页数量 |

### GET /api/products/{product_id}

商品详情。访问该接口时，登录用户需要记录 `view` 行为。

### GET /api/products/hot

热门商品列表。用于首页兜底推荐。

## 4. 购物车接口

### GET /api/cart

获取当前用户购物车。

### POST /api/cart/items

加入购物车。

请求：

```json
{
  "product_id": 1,
  "quantity": 2
}
```

行为记录：写入 `cart` 行为。

### PUT /api/cart/items/{cart_item_id}

修改购物车数量。

请求：

```json
{
  "quantity": 3,
  "selected": true
}
```

### DELETE /api/cart/items/{cart_item_id}

删除购物车项。

## 5. 订单接口

### POST /api/orders

提交订单。

请求：

```json
{
  "cart_item_ids": [1, 2],
  "receiver_name": "张三",
  "receiver_phone": "13800000000",
  "receiver_address": "北京市朝阳区",
  "remark": "尽快发货"
}
```

响应：

```json
{
  "order_id": 1,
  "order_no": "202606220001",
  "total_amount": 199.00,
  "status": "pending"
}
```

行为记录：对订单内商品写入 `purchase` 行为。

### GET /api/orders

当前用户订单列表。

### GET /api/orders/{order_id}

当前用户订单详情。

### PUT /api/orders/{order_id}/cancel

取消订单。仅允许取消未发货订单。

## 8. AI Agent 接口

### POST /api/ai/chat

统一 AI 对话入口。

请求：

```json
{
  "agent_type": "customer",
  "conversation_id": 1,
  "message": "我的订单什么时候发货？"
}
```

`agent_type` 可选：

```text
customer
recommendation
operation
inventory
```

响应：

```json
{
  "conversation_id": 1,
  "agent_type": "customer",
  "answer": "我查到你的订单目前是待处理状态，商家会尽快发货。",
  "tool_calls": [
    {
      "tool_name": "get_user_orders",
      "status": "success"
    }
  ]
}
```

### GET /api/ai/conversations

获取当前用户或管理员的 AI 会话列表。

### GET /api/ai/conversations/{conversation_id}/messages

获取会话消息。

### GET /api/admin/ai/tool-calls

后台查看 Agent 工具调用日志。

## 10. 2026-06-23 更新：管理员商品图片上传接口

### POST /api/admin/uploads/product-image

管理员上传商品主图。该接口只能由管理员调用，普通用户和未登录请求会被拒绝。

请求类型：

```text
multipart/form-data
```

请求字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| file | File | 是 | 商品图片文件 |

支持格式：

```text
jpg, jpeg, png, webp, gif
```

大小限制：

```text
最大 5MB
```

成功响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "url": "/uploads/products/uuid.jpg",
    "filename": "uuid.jpg",
    "size": 12345,
    "content_type": "image/jpeg"
  }
}
```

使用方式：

- 上传成功后，将 `data.url` 写入商品的 `main_image` 字段。
- `POST /api/admin/products` 和 `PUT /api/admin/products/{product_id}` 继续使用 `main_image` 保存商品主图地址。
- `main_image` 可以是 `/uploads/products/xxx.jpg`，也可以是外链 `https://...`。

常见错误：

| HTTP 状态 | 场景 |
| --- | --- |
| 400 | 文件为空、格式不支持、超过 5MB |
| 401 | 未登录或不是管理员 token |
