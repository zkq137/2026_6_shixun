# 队员本地运行指南

本文档给项目组成员使用，目标是让每个人都能在自己电脑上安装环境、初始化数据库、启动前后端，并避免把本地环境文件提交到 Git。

## 1. 必备环境

请先安装并确认以下环境可用：

| 环境 | 推荐版本 | 检查命令 |
| --- | --- | --- |
| Git | 2.x | `git --version` |
| Python | 3.11 或 3.12 | `python --version` |
| Node.js | 20.x 或 22.x | `node --version` |
| npm | 随 Node 安装 | `npm --version` |
| MySQL | 8.0 | `mysql --version` |

如果 PowerShell 无法激活 Python 虚拟环境，先执行一次：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

## 2. 拉取项目

```powershell
git clone <项目仓库地址>
cd 2026_6_融创软通大模型
```

如果项目路径包含中文，一般可以正常运行；如果某个队员环境出现编码或路径问题，可以把项目放到纯英文路径，例如：

```text
C:\projects\ai-mall
```

## 3. Git 提交注意事项

项目已经配置 `.gitignore`，以下内容不要提交：

```text
backend/.venv/
frontend/node_modules/
frontend/.npm-cache/
frontend/dist/
backend/.env
*.log
*.tsbuildinfo
.agents/
.codex/
.vscode/
.idea/
```

提交前先检查：

```powershell
git status --short
```

正常应该只看到源码、文档、配置模板等需要提交的文件。不要看到 `.venv`、`node_modules`、`.env`、`.npm-cache`、`dist` 这类文件。

如果不小心看到这些文件已经被 Git 跟踪，不要直接删除本地文件，使用：

```powershell
git rm -r --cached backend/.venv
git rm -r --cached frontend/node_modules
git rm -r --cached frontend/.npm-cache
git rm -r --cached frontend/dist
git rm --cached backend/.env
```

然后重新执行：

```powershell
git status --short
```

## 4. 后端环境配置

进入后端目录：

```powershell
cd backend
```

创建虚拟环境：

```powershell
python -m venv .venv
```

激活虚拟环境：

```powershell
.\.venv\Scripts\Activate.ps1
```

安装依赖：

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

复制环境变量模板：

```powershell
Copy-Item .env.example .env
```

编辑 `backend/.env`，至少修改数据库密码：

```env
DATABASE_URL=mysql+pymysql://root:你的MySQL密码@localhost:3306/ai_mall?charset=utf8mb4
```

如果要测试 AI Agent 的真实大模型能力，再配置 DeepSeek：

```env
LLM_PROVIDER=openai_compatible
LLM_BASE_URL=https://api.deepseek.com
LLM_API_KEY=你的DeepSeekKey
LLM_MODEL=deepseek-v4-pro
LLM_TIMEOUT_SECONDS=30
```

注意：`backend/.env` 不能提交到 Git。

## 5. MySQL 初始化

确认 MySQL 服务已启动。Windows 常见命令：

```powershell
net start MySQL80
```

登录 MySQL：

```powershell
mysql -u root -p
```

创建数据库：

```sql
CREATE DATABASE ai_mall DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

如果已经存在，可以跳过创建数据库。

退出 MySQL：

```sql
exit;
```

初始化表和演示数据：

```powershell
.\.venv\Scripts\python.exe scripts\init_db.py
```

检查数据库：

```powershell
.\.venv\Scripts\python.exe scripts\check_db.py
```

## 6. 启动后端

在 `backend` 目录执行：

```powershell
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

启动成功后访问：

```text
http://127.0.0.1:8000/docs
```

可以先测试健康检查：

```text
http://127.0.0.1:8000/api/health
http://127.0.0.1:8000/api/health/db
```

后端窗口不要关闭，前端需要连接它。

## 7. 前端环境配置

新开一个 PowerShell 窗口，进入前端目录：

```powershell
cd frontend
```

安装依赖：

```powershell
npm install
```

确认 `frontend/.env.development` 内容为：

```env
VITE_API_BASE_URL=http://127.0.0.1:8000/api
```

启动前端：

```powershell
npm run dev -- --host 127.0.0.1 --port 5174
```

访问：

```text
http://127.0.0.1:5174
```

如果 5174 被占用，可以换端口：

```powershell
npm run dev -- --host 127.0.0.1 --port 5175
```

## 8. 默认账号

管理员账号：

```text
用户名：admin
密码：admin123456
```

普通用户可以在前端注册，也可以使用初始化数据中的演示用户，具体以 `scripts/init_db.py` 中的数据为准。

## 9. 常见问题

### 后端提示 MySQL 连接失败

先确认 MySQL 服务是否启动：

```powershell
net start MySQL80
```

再确认 `backend/.env` 里的 `DATABASE_URL` 密码、端口、数据库名是否正确。

### 前端提示 Network Error

通常是后端没启动，或者前端 API 地址不对。检查：

```text
http://127.0.0.1:8000/api/health
```

如果打不开，先启动后端。

### AI 提示 LLM service unavailable

先检查 `backend/.env`：

```env
LLM_BASE_URL=https://api.deepseek.com
LLM_API_KEY=你的DeepSeekKey
LLM_MODEL=deepseek-v4-pro
```

改完 `.env` 后需要重启后端。

### PowerShell 禁止运行 Activate.ps1

执行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

然后重新激活：

```powershell
.\.venv\Scripts\Activate.ps1
```

```powershell
git pull
```

启动 MySQL。

启动后端：

```powershell
cd backend
.\.venv\Scripts\Activate.ps1
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

启动前端：

```powershell
cd frontend
npm run dev -- --host 127.0.0.1 --port 5174
```

提交前检查：

```powershell
git status --short
```

确认没有 `.venv`、`node_modules`、`.env`、`.npm-cache`、`dist` 后再提交。
