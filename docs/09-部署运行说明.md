# 部署运行说明

## 1. 环境要求

推荐开发环境：

- Python 3.11+
- Node.js 20+
- MySQL 8.0+
- Redis 7.x，可选
- Git

推荐端口：

| 服务 | 端口 |
| --- | --- |
| Vue3 前端 | 5173 |
| FastAPI 后端 | 8000 |
| MySQL | 3306 |
| Redis | 6379 |

## 2. 后端启动

进入后端目录：

```bash
cd backend
```

创建虚拟环境：

```bash
python -m venv .venv
```

激活虚拟环境：

Windows PowerShell：

```powershell
.\.venv\Scripts\Activate.ps1
```

安装依赖：

```bash
pip install -r requirements.txt
```

启动服务：

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

健康检查：

```text
http://localhost:8000/health
```

接口文档：

```text
http://localhost:8000/docs
```

## 3. 前端启动

进入前端目录：

```bash
cd frontend
```

安装依赖：

```bash
npm install
```

启动开发服务：

```bash
npm run dev
```

访问：

```text
http://localhost:5173
```

## 4. 数据库配置

创建数据库：

```sql
CREATE DATABASE ai_mall DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

推荐数据库连接环境变量：

```text
DATABASE_URL=mysql+pymysql://root:password@localhost:3306/ai_mall
```

如果使用 Alembic 管理迁移：

```bash
alembic upgrade head
```

## 5. Redis 配置

Redis 为可选依赖，可用于：

- 热门商品缓存。
- 推荐结果缓存。
- AI 会话临时上下文缓存。

环境变量：

```text
REDIS_URL=redis://localhost:6379/0
```

Redis 不可用时，系统核心业务必须仍可运行。

## 6. 环境变量

后端建议使用 `.env`：

```text
APP_NAME=AI智能商城系统
APP_ENV=development
SECRET_KEY=replace-with-random-secret
ACCESS_TOKEN_EXPIRE_MINUTES=1440

DATABASE_URL=mysql+pymysql://root:password@localhost:3306/ai_mall
REDIS_URL=redis://localhost:6379/0

LLM_PROVIDER=openai_compatible
LLM_BASE_URL=https://api.example.com/v1
LLM_API_KEY=replace-with-your-api-key
LLM_MODEL=deepseek-chat
```

前端建议使用 `.env.development`：

```text
VITE_API_BASE_URL=http://localhost:8000/api
```

## 7. 大模型 API 配置

系统通过统一 `LLMProvider` 调用大模型。后续实现时应避免在业务代码中直接写死供应商。

推荐配置项：

| 变量 | 说明 |
| --- | --- |
| LLM_PROVIDER | 模型供应商类型 |
| LLM_BASE_URL | API 地址 |
| LLM_API_KEY | API Key |
| LLM_MODEL | 模型名称 |
| LLM_TIMEOUT_SECONDS | 超时时间 |

大模型不可用时：

- 客服 Agent 使用 FAQ 兜底。
- 推荐 Agent 返回热门商品。
- 运营 Agent 返回工具数据摘要。
- 库存 Agent 使用移动平均预测。

## 8. 本地演示数据

建议准备：

- 商品分类：数码、服饰、家居。
- 商品：每类至少 3 个。
- 用户：普通用户 2 个。
- 管理员：至少 1 个。
- FAQ：退货、换货、物流、订单、商品咨询。
- 订单：至少 5 条。
- 用户行为：浏览、加购、购买共至少 20 条。
- 销量统计：至少 7 天数据。

## 9. 常见问题

### 9.1 后端无法连接数据库

检查：

- MySQL 是否启动。
- `DATABASE_URL` 是否正确。
- 数据库用户和密码是否正确。
- 数据库是否已创建。

### 9.2 前端接口请求失败

检查：

- 后端是否启动。
- `VITE_API_BASE_URL` 是否正确。
- 后端是否允许跨域。

### 9.3 AI 功能不可用

检查：

- `LLM_API_KEY` 是否配置。
- `LLM_BASE_URL` 是否可访问。
- 模型名称是否正确。
- 是否触发了降级策略。

### 9.4 推荐结果为空

检查：

- 是否有上架商品。
- 用户是否有行为记录。
- 热门商品接口是否可用。
- 推荐失败时是否返回热门商品兜底。
