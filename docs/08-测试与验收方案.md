# 测试与验收方案

## 1. 测试目标

- 验证电商基础流程完整可用。
- 验证后台管理功能可用。
- 验证 AI Agent 能通过工具调用完成任务。
- 验证异常情况下系统能给出明确提示或降级结果。

## 2. 功能测试

### 2.1 用户认证

测试用例：

- 注册新用户成功。
- 重复用户名注册失败。
- 正确账号密码登录成功。
- 错误密码登录失败。
- 未登录访问购物车失败。

验收标准：

- token 能正确生成和校验。
- 用户只能访问自己的数据。

### 2.2 商品

测试用例：

- 查看商品列表。
- 按关键词搜索商品。
- 按分类筛选商品。
- 查看商品详情。
- 下架商品不在用户端展示。

### 2.2.1 商品评论

测试用例：

- 未登录用户可以查看评论列表。
- 未登录用户不能发表评论。
- 登录用户可以提交评分和评论内容。
- 匿名评论在用户端显示为匿名用户。
- 管理员可以隐藏和恢复评论。
- 被隐藏评论不在用户端展示。
- 已有评论的商品不能永久删除。

验收标准：

- 评论分页、评分汇总和已购买标识展示正确。
- 后台评论状态修改能同步影响用户端展示。

验收标准：

- 搜索、筛选、分页结果正确。
- 商品价格、库存、状态展示准确。

### 2.3 购物车

测试用例：

- 加入购物车。
- 重复加入同一商品。
- 修改商品数量。
- 删除购物车商品。
- 库存不足时禁止超量提交。

验收标准：

- 同一商品数量累加。
- 总价计算准确。

### 2.4 订单

测试用例：

- 从购物车创建订单。
- 创建订单后扣减库存。
- 查看我的订单。
- 查看订单详情。
- 取消未发货订单。
- 用户访问他人订单失败。

验收标准：

- 订单主表和明细数据完整。
- 库存扣减和恢复正确。
- 权限隔离正确。

### 2.5 后台管理

测试用例：

- 管理员登录。
- 新增商品。
- 编辑商品。
- 上架和下架商品。
- 修改订单状态。
- 禁用用户。

验收标准：

- 普通用户不能访问后台接口。
- 后台修改能同步影响用户端展示。

## 3. 接口测试

建议使用 pytest + httpx。

重点接口：

- `POST /api/auth/register`
- `POST /api/auth/login`
- `GET /api/products`
- `GET /api/products/{product_id}`
- `POST /api/cart/items`
- `POST /api/orders`
- `GET /api/orders`
- `POST /api/ai/chat`
- `GET /api/admin/ai/tool-calls`

接口测试要求：

- 正常请求返回 `code = 0`。
- 参数错误返回 400。
- 未登录返回 401。
- 权限不足返回 403。
- 业务冲突返回 409。

## 4. AI 功能测试

### 4.1 客服 Agent

场景：

```text
用户问：怎么退货？
```

预期：

- 调用 `query_faq`。
- 返回退货规则。
- 记录工具调用日志。

场景：

```text
用户问：我的订单什么时候发货？
```

预期：

- 调用 `get_user_orders` 或 `get_order_detail`。
- 返回订单状态。
- 不编造物流信息。

### 4.2 推荐 Agent

场景：

```text
登录用户访问首页推荐。
```

预期：

- 调用 `get_user_behavior`。
- 调用 `get_hot_products` 或 `get_similar_products`。
- 返回商品和推荐理由。

场景：

```text
新用户访问推荐。
```

预期：

- 返回热门商品兜底。

### 4.3 运营 Agent

场景：

```text
管理员问：最近7天哪些商品销量最好？
```

预期：

- 调用 `get_sales_statistics`。
- 返回热销商品排行。
- 包含数据依据和运营建议。

### 4.4 库存 Agent

场景：

```text
管理员问：帮我看看哪些商品库存有风险。
```

预期：

- 调用 `predict_product_sales`。
- 调用 `check_inventory_risk`。
- 必要时调用 `create_inventory_alert`。
- 返回风险等级和补货建议。

## 5. Agent 工具调用测试

每次 Agent 调用工具后，检查 `agent_tool_calls`：

- `conversation_id` 正确。
- `agent_type` 正确。
- `tool_name` 正确。
- `input_json` 有参数。
- `output_json` 有结果。
- `status` 为 success 或 failed。
- `duration_ms` 有值。

权限测试：

- 用户不能通过客服 Agent 查询其他用户订单。
- 普通用户不能调用运营 Agent 和库存 Agent。
- 工具不能返回密码、token、密钥等敏感字段。

## 6. 降级测试

测试场景：

- 大模型 API Key 缺失。
- 大模型接口调用失败。
- 推荐计算无结果。
- 销量数据不足。
- Redis 不可用。

预期：

- 客服返回 FAQ 或兜底提示。
- 推荐返回热门商品。
- 预测使用最近 7 日平均销量。
- 核心购物流程不受 Redis 影响。

## 7. 演示验收清单

演示前必须准备：

- 至少 10 个商品。
- 至少 3 个分类。
- 至少 2 个用户。
- 至少 5 条订单。
- 至少 20 条用户行为记录。
- 至少 8 条 FAQ。
- 至少 7 天销量统计数据。
- 至少 2 条库存预警。

演示流程：

1. 用户登录。
2. 浏览商品并加入购物车。
3. 提交订单。
4. 查看推荐商品。
5. 使用客服 Agent 查询退货或订单。
6. 管理员登录后台。
7. 查看商品和订单管理。
8. 使用运营 Agent 分析热销商品。
9. 使用库存 Agent 生成库存建议。
10. 查看 Agent 工具调用日志。

## 8. 最终验收标准

- 用户端购物闭环可完成。
- 后台管理功能可完成。
- 4 个 Agent 至少各有一个可演示场景。
- Agent 工具调用日志可查询。
- 推荐、客服、运营、库存功能均有降级策略。
- 文档、接口和实际实现保持一致。


## 2026-06-23 更新：商品图片上传验收

新增测试项：

- 未登录访问 `POST /api/admin/uploads/product-image` 返回 401。
- 普通用户 token 访问上传接口返回 401。
- 管理员上传 jpg/png/webp/gif 图片成功。
- 上传非图片文件失败。
- 上传超过 5MB 文件失败。
- 上传成功后返回 `/uploads/products/...` 地址。
- 商品保存该地址后，首页、商品列表、商品详情能显示图片。
- 图片文件不存在或加载失败时，前端显示商品名前两个字兜底。
