# 开发计划

## 阶段一：基础工程搭建

目标：建立前后端项目骨架和数据库连接。

任务：

- 创建 FastAPI 后端项目。
- 创建 Vue3 + Vite 前端项目。
- 配置 MySQL 数据库连接。
- 配置环境变量读取。
- 建立统一响应格式和异常处理。
- 建立 JWT 登录认证基础能力。

交付物：

- 后端可以启动并访问健康检查接口。
- 前端可以启动并访问首页。
- 数据库连接成功。
- 登录接口初步可用。

## 阶段二：电商核心业务

目标：完成用户端购物闭环。

任务：

- 用户注册、登录、个人信息。
- 商品分类。
- 商品列表、搜索、详情。
- 购物车新增、修改、删除。
- 创建订单、查看订单、取消订单。
- 下单时扣减库存。
- 浏览、加购、购买行为记录。

交付物：

- 用户可以完成浏览商品到提交订单的完整流程。
- 数据库中有订单、订单明细和用户行为记录。

## 阶段三：后台管理

目标：完成管理员基础管理能力。

任务：

- 管理员登录。
- 后台商品管理。
- 后台分类管理。
- 后台订单管理。
- 后台用户管理。
- 后台库存查看。
- 后台首页看板。

交付物：

- 管理员可以维护商品、分类和订单。
- 后台首页可以展示基础统计数据。

## 阶段四：AI推荐与客服

目标：完成第一批 AI 亮点。

任务：

- 建立 FAQ 数据表和查询接口。
- 实现客服 Agent 的 FAQ 查询能力。
- 实现用户行为查询。
- 实现热门推荐。
- 实现看了又看。
- 实现推荐 Agent 生成推荐理由。
- 增加 AI 会话和消息记录。

交付物：

- 用户可以使用 AI 客服咨询常见问题。
- 首页和详情页可以展示推荐商品。
- 推荐结果带有推荐理由。

## 阶段五：多 Agent 工具调用

目标：突出 Agent 调用工具工作的能力。

任务：

- 建立 Tool Registry。
- 实现工具调用统一输入输出。
- 实现 Agent 工具调用日志。
- 完善客服 Agent 调用订单和商品工具。
- 实现运营 Agent。
- 实现库存 Agent。
- 实现销量统计工具。
- 实现销量预测工具。
- 实现库存风险检查和预警创建。

交付物：

- 后台 AI 运营助手可回答销售分析问题。
- 库存 Agent 可以生成库存风险建议。
- 后台可以查看 Agent 工具调用日志。

## 阶段六：测试、演示、答辩准备

目标：确保系统稳定可演示。

任务：

- 准备演示商品、用户、订单、FAQ 和行为数据。
- 编写接口测试。
- 验证用户端购物流程。
- 验证后台管理流程。
- 验证 4 个 Agent 的核心场景。
- 准备答辩 PPT 和演示脚本。

交付物：

- 系统可稳定本地运行。
- 有完整演示数据。
- 有测试与验收记录。

## 推荐时间安排

| 阶段 | 建议时长 |
| --- | --- |
| 阶段一 | 1-2 天 |
| 阶段二 | 3-5 天 |
| 阶段三 | 2-3 天 |
| 阶段四 | 2-3 天 |
| 阶段五 | 3-5 天 |
| 阶段六 | 1-2 天 |

## 风险与应对

| 风险 | 应对 |
| --- | --- |
| AI 接口不稳定 | FAQ、热门推荐、移动平均预测作为兜底 |
| Agent 过于复杂 | 固定 4 个 Agent，工具清单固定 |
| 数据不足导致推荐效果差 | 准备演示行为数据 |
| 时间不足 | 优先保证电商闭环和客服、推荐功能 |
| 前后端联调慢 | 先用接口文档和 Mock 数据并行开发 |

## 最小可演示版本

如果时间紧张，至少完成：

- 用户登录注册。
- 商品浏览、搜索、详情。
- 购物车和订单。
- 后台商品和订单管理。
- FAQ 客服 Agent。
- 热门推荐和看了又看。
- 运营 Agent 或库存 Agent 至少完成一个可演示工具调用链。
