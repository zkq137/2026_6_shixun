# 系统架构设计

## 1. 总体架构

系统采用前后端分离架构，后端提供 REST API 和 AI Agent 服务，前端负责用户商城、管理后台和 AI 交互界面。

```text
Vue3 前端
  |
  | HTTP / JSON
  v
FastAPI 后端
  |
  +-- 业务服务层
  +-- AI Agent 编排层
  +-- Agent 工具层
  |
  +-- MySQL
  +-- Redis 可选
  +-- 大模型 API
```

## 2. 后端架构

推荐目录：

```text
backend/
├── app/
│   ├── api/
│   ├── agents/
│   ├── core/
│   ├── models/
│   ├── repositories/
│   ├── schemas/
│   ├── services/
│   ├── tools/
│   └── main.py
├── tests/
└── requirements.txt
```

分层职责：

| 目录 | 职责 |
| --- | --- |
| api | 路由、参数接收、权限校验入口 |
| schemas | Pydantic 请求和响应模型 |
| services | 业务逻辑，例如下单、推荐、预测 |
| repositories | 数据访问封装 |
| models | SQLAlchemy 数据模型 |
| agents | 多 Agent 编排、提示词、调用流程 |
| tools | Agent 可调用工具 |
| core | 配置、认证、异常、日志、安全 |

## 3. 前端架构

推荐目录：

```text
frontend/
├── src/
│   ├── api/
│   ├── assets/
│   ├── components/
│   ├── layouts/
│   ├── router/
│   ├── stores/
│   ├── views/
│   └── main.ts
└── package.json
```

前端分为两个主要区域：

- 用户商城：商品浏览、购物车、订单、AI 客服、推荐。
- 管理后台：商品管理、订单管理、库存管理、AI 运营助手、预测看板。

## 4. AI Agent 架构

Agent 不直接访问数据库，统一通过工具层完成查询和操作。

```text
用户或管理员问题
  |
  v
Agent Router
  |
  +-- 客服 Agent
  +-- 推荐 Agent
  +-- 运营 Agent
  +-- 库存 Agent
  |
  v
Tool Registry
  |
  +-- 商品工具
  +-- 订单工具
  +-- FAQ 工具
  +-- 行为工具
  +-- 销售统计工具
  +-- 库存预测工具
```

Agent Router 根据入口和问题类型选择 Agent：

| 入口 | 默认 Agent |
| --- | --- |
| 用户客服窗口 | 客服 Agent |
| 商品推荐区 | 推荐 Agent |
| 后台 AI 运营助手 | 运营 Agent |
| 库存预警页面 | 库存 Agent |

## 5. 核心业务数据流

### 5.1 下单流程

```text
用户提交订单
  -> 校验登录状态
  -> 查询购物车
  -> 校验商品状态和库存
  -> 创建订单主表
  -> 创建订单明细
  -> 扣减库存
  -> 记录购买行为
  -> 返回订单结果
```

### 5.2 推荐流程

```text
用户进入首页或详情页
  -> 查询用户行为
  -> 查询相似商品或热门商品
  -> 过滤下架和库存不足商品
  -> 推荐 Agent 生成推荐理由
  -> 返回推荐列表
```

### 5.3 客服流程

```text
用户输入问题
  -> 客服 Agent 判断意图
  -> 优先调用 query_faq
  -> 需要订单信息时调用订单工具
  -> FAQ 未命中时调用大模型
  -> 保存会话和工具调用日志
  -> 返回回答
```

### 5.4 库存预测流程

```text
管理员发起库存分析
  -> 库存 Agent 查询库存
  -> 调用销量统计工具
  -> 调用销量预测工具
  -> 判断库存风险
  -> 必要时创建库存预警
  -> 返回补货建议
```

## 6. 部署结构

开发环境：

```text
localhost:5173  Vue3 前端
localhost:8000  FastAPI 后端
localhost:3306  MySQL
localhost:6379  Redis 可选
```

生产或演示环境可采用：

```text
Nginx
  +-- 静态前端
  +-- 反向代理 /api 到 FastAPI
FastAPI + Uvicorn/Gunicorn
MySQL
Redis 可选
```

## 7. 关键设计约束

- 后端接口统一返回 JSON。
- 用户端和管理端共用后端服务，但权限分离。
- Agent 工具必须复用已有 Service，不重复写数据库逻辑。
- Agent 工具调用必须记录日志。
- 推荐、预测、客服应有降级策略，避免大模型不可用导致系统不可演示。
