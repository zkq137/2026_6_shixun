# 前端页面设计

## 1. 前端技术栈

- Vue3
- Vite
- TypeScript
- Pinia
- Vue Router
- Element Plus
- Axios

## 2. 页面分区

系统前端分为用户商城和管理后台两个区域。

```text
/                 用户商城
/admin            管理后台
```

## 3. 用户端页面

### 3.1 首页

路由：`/`

功能：

- 展示商品分类。
- 展示热门商品。
- 展示个性化推荐。
- 提供搜索入口。
- 提供 AI 客服入口。

数据来源：

- `GET /api/categories`
- `GET /api/products/hot`
- `GET /api/recommendations?scene=home`

### 3.2 商品列表页

路由：`/products`

功能：

- 根据关键词、分类、价格区间筛选商品。
- 商品分页展示。
- 商品卡片展示图片、名称、价格、销量。

数据来源：

- `GET /api/products`

### 3.3 商品详情页

路由：`/products/:id`

功能：

- 展示商品主图、名称、价格、库存、销量、描述。
- 加入购物车。
- 展示“看了又看”推荐。
- 可打开客服咨询商品。

数据来源：

- `GET /api/products/{product_id}`
- `GET /api/products/{product_id}/similar`
- `POST /api/cart/items`

### 3.4 购物车页

路由：`/cart`

功能：

- 展示购物车商品。
- 修改数量。
- 删除商品。
- 选择商品结算。
- 展示总价。

数据来源：

- `GET /api/cart`
- `PUT /api/cart/items/{cart_item_id}`
- `DELETE /api/cart/items/{cart_item_id}`

### 3.5 订单确认页

路由：`/checkout`

功能：

- 展示待结算商品。
- 填写收货信息。
- 提交订单。

数据来源：

- `POST /api/orders`

### 3.6 我的订单页

路由：`/orders`

功能：

- 展示当前用户订单列表。
- 查看订单详情。
- 取消未发货订单。

数据来源：

- `GET /api/orders`
- `GET /api/orders/{order_id}`
- `PUT /api/orders/{order_id}/cancel`

### 3.7 个人中心

路由：`/profile`

功能：

- 查看个人信息。
- 修改昵称、手机号等信息。

数据来源：

- `GET /api/users/me`

### 3.8 AI 客服页

路由：`/ai/customer-service`

功能：

- 用户输入问题。
- 展示多轮对话。
- 展示 Agent 正在查询的状态。
- 对订单类问题支持自动查询本人订单。

数据来源：

- `POST /api/ai/chat`
- `GET /api/ai/conversations`
- `GET /api/ai/conversations/{conversation_id}/messages`

## 4. 管理后台页面

### 4.1 后台登录页

路由：`/admin/login`

功能：

- 管理员登录。

数据来源：

- `POST /api/admin/auth/login`

### 4.2 后台首页

路由：`/admin/dashboard`

功能：

- 展示今日销售额。
- 展示今日订单数。
- 展示用户总数。
- 展示库存预警数量。
- 展示近期销量趋势。

数据来源：

- `GET /api/admin/dashboard`
- `GET /api/admin/sales/statistics`

### 4.3 商品管理

路由：`/admin/products`

功能：

- 商品列表。
- 新增商品。
- 编辑商品。
- 上架和下架。
- 删除商品。

数据来源：

- `GET /api/admin/products`
- `POST /api/admin/products`
- `PUT /api/admin/products/{product_id}`
- `PUT /api/admin/products/{product_id}/status`
- `DELETE /api/admin/products/{product_id}`

### 4.4 分类管理

路由：`/admin/categories`

功能：

- 分类列表。
- 新增分类。
- 编辑分类。
- 启用或禁用分类。

### 4.5 订单管理

路由：`/admin/orders`

功能：

- 查看所有订单。
- 按状态、时间、用户筛选。
- 修改订单状态。
- 查看订单详情。

数据来源：

- `GET /api/admin/orders`
- `PUT /api/admin/orders/{order_id}/status`

### 4.6 用户管理

路由：`/admin/users`

功能：

- 查看用户列表。
- 禁用或启用用户。

数据来源：

- `GET /api/admin/users`
- `PUT /api/admin/users/{user_id}/status`

### 4.7 库存管理

路由：`/admin/inventory`

功能：

- 查看商品库存。
- 查看库存风险。
- 处理库存预警。
- 触发库存 Agent 分析。

数据来源：

- `GET /api/admin/inventory/alerts`
- `PUT /api/admin/inventory/alerts/{alert_id}/status`
- `POST /api/ai/chat`

### 4.8 AI 运营助手

路由：`/admin/ai/operation`

功能：

- 管理员用自然语言提问。
- 运营 Agent 分析销量、商品和用户行为。
- 展示 Agent 调用过的工具。

示例问题：

```text
最近7天哪些商品销量最好？
哪些商品浏览很多但购买少？
帮我找出适合促销的商品。
```

数据来源：

- `POST /api/ai/chat`
- `GET /api/admin/ai/tool-calls`

### 4.9 销量预测页

路由：`/admin/sales-prediction`

功能：

- 选择商品。
- 查看历史销量。
- 生成未来销量预测。
- 查看预测依据。

数据来源：

- `GET /api/admin/sales/statistics`
- `POST /api/admin/sales/predict`

## 6. 交互要求

- 用户端优先保证购买流程清晰。
- 管理端优先保证表格、筛选、编辑效率。
- AI 对话页面需要展示工具调用状态，例如“正在查询订单”。
- 后台 AI 页面需要展示 Agent 结论、数据依据和建议动作。
- 请求失败时需要给出清晰错误提示。


## 7. 2026-06-23 更新：商品图片管理

后台商品管理页 `/admin/products` 已支持管理员直接管理商品主图。

新增交互：

- 商品列表展示主图缩略图。
- 新增商品时可上传主图。
- 编辑商品时可预览、替换、清空主图。
- 上传成功后自动把返回的图片地址写入 `main_image`。
- 保留手动填写图片路径或外链 URL 的能力。

图片地址规则：

```text
/uploads/products/xxx.jpg
https://example.com/xxx.jpg
```

用户端展示规则：

- 首页、商品列表、商品详情优先显示 `main_image`。
- 图片加载失败时显示商品名前两个字作为兜底占位。

前端接口：

```text
POST /api/admin/uploads/product-image
```

上传字段：

```text
file
```
