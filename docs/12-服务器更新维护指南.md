# 服务器更新维护指南

本文档用于项目已经部署到阿里云服务器之后，指导如何把本地最新代码更新到服务器。

当前服务器信息：

```text
公网 IP：39.107.99.7
项目目录：/opt/ai-mall
后端服务：ai-mall-backend
前端目录：/opt/ai-mall/frontend
后端目录：/opt/ai-mall/backend
```

## 1. 更新代码的整体流程

每次更新服务器，一般按这个顺序：

```text
1. 本地提交代码并推送到 GitHub
2. 登录服务器
3. 在服务器执行 git pull
4. 如有后端依赖变化，重新安装 requirements.txt
5. 重启后端 systemd 服务
6. 如有前端变化，重新 npm install 和 npm run build
7. 检查并重载 Nginx
8. 浏览器强制刷新页面
```

## 2. 本地电脑操作

在本地项目目录执行：

```powershell
git status
git add .
git commit -m "说明你改了什么"
git push origin main
```

如果项目已经启用了 PR 规则，组员不要直接推 `main`，而是：

```powershell
git checkout -b feature/功能名
git add .
git commit -m "说明你改了什么"
git push origin feature/功能名
```

然后在 GitHub 创建 Pull Request，审核通过后合并到 `main`。

## 3. 登录服务器

在本地 PowerShell 执行：

```powershell
ssh admin@39.107.99.7
```

登录后进入项目目录：

```bash
cd /opt/ai-mall
```

## 4. 拉取最新代码

```bash
git pull
```

如果提示本地有修改，说明有人直接在服务器改过文件。正常开发不建议这样做，应尽量通过 GitHub 更新服务器。

## 5. 更新后端

如果只改了后端 Python 代码，执行：

```bash
sudo systemctl restart ai-mall-backend
sudo systemctl status ai-mall-backend
```

看到下面状态说明成功：

```text
active (running)
```

如果修改了 `backend/requirements.txt`，还要重新安装依赖：

```bash
cd /opt/ai-mall/backend
source .venv/bin/activate
pip install -r requirements.txt
sudo systemctl restart ai-mall-backend
```

如果安装很慢，可以使用清华源：

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

## 6. 更新数据库

一般情况下，不要随便重新初始化数据库。

只有以下情况才考虑执行：

```text
新增了表
新增了必要字段
更新了初始化脚本
需要补充演示数据
```

执行命令：

```bash
cd /opt/ai-mall/backend
source .venv/bin/activate
python scripts/init_db.py
python scripts/check_db.py
```

注意：当前项目没有使用 Alembic 迁移系统，修改表结构前应先确认脚本是否幂等，避免破坏已有数据。

## 7. 更新前端

如果改了 `frontend/src`、路由、页面、API 调用等前端内容，需要重新打包：

```bash
cd /opt/ai-mall/frontend
npm install
npm run build
```

前端生产环境需要保证 `frontend/.env.production` 存在，并且内容是：

```env
VITE_API_BASE_URL=/api
```

这样浏览器访问公网地址时，请求会走 Nginx 的 `/api` 反向代理，而不是请求本地电脑的 `localhost`。

## 8. 重载 Nginx

前端打包完成后，通常不需要重启 Nginx，但建议检查配置并重载：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

如果 `nginx -t` 输出下面内容，说明配置正常：

```text
syntax is ok
test is successful
```

## 9. 常用完整更新命令

如果不确定改了前端还是后端，可以直接执行完整流程：

```bash
cd /opt/ai-mall
git pull

cd backend
source .venv/bin/activate
pip install -r requirements.txt
sudo systemctl restart ai-mall-backend

cd ../frontend
npm install
npm run build

sudo nginx -t
sudo systemctl reload nginx
```

然后浏览器打开：

```text
http://39.107.99.7
```

如果页面没有变化，按：

```text
Ctrl + F5
```

强制刷新浏览器缓存。

## 10. 按修改类型选择命令

| 修改内容 | 服务器需要做什么 |
| --- | --- |
| 只改后端 Python 代码 | `git pull` 后重启 `ai-mall-backend` |
| 只改前端 Vue 页面 | `git pull` 后执行 `npm run build` |
| 改了后端依赖 | 重新 `pip install -r requirements.txt` |
| 改了前端依赖 | 重新 `npm install` |
| 改了 `.env` | 重启 `ai-mall-backend` |
| 改了 Nginx 配置 | `sudo nginx -t` 后 `sudo systemctl reload nginx` |
| 改了数据库结构 | 谨慎执行初始化或迁移脚本 |

## 11. 查看服务状态和日志

查看后端状态：

```bash
sudo systemctl status ai-mall-backend
```

实时查看后端日志：

```bash
sudo journalctl -u ai-mall-backend -f
```

查看 Nginx 状态：

```bash
sudo systemctl status nginx
```

查看 Nginx 错误日志：

```bash
sudo tail -f /var/log/nginx/error.log
```

查看 Nginx 访问日志：

```bash
sudo tail -f /var/log/nginx/access.log
```

## 12. 注意事项

- 不要在服务器上直接开发代码，正常流程是本地开发、GitHub 合并、服务器 `git pull`。
- 不要把 `backend/.env` 提交到 GitHub。
- 不要开放 MySQL 的 `3306` 端口到公网。
- 修改 DeepSeek API Key 后，需要重启后端服务。
- 前端打包后如果仍然请求本地地址，检查 `frontend/.env.production` 是否为 `VITE_API_BASE_URL=/api`。
- 如果服务器页面异常，先检查后端日志和浏览器开发者工具 Network 面板。
